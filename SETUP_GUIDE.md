# 顧客管理システム セットアップ手順(友人向け)

このシステム(Next.js + Vercel + Neon Postgres)を、あなた専用の環境として使い始めるための手順です。パソコンの種類やネット環境を問わず、ブラウザの操作だけで完結します。ターミナルやコマンド入力は不要です。

## 事前に用意するもの

- GitHubの招待メールを承認済みであること(まだの場合は招待メールから承認してください)
- 無料のVercelアカウント(なければ手順の中で作成します)

## 手順

1. [vercel.com](https://vercel.com) にアクセスし、アカウントを作成(またはログイン)する
2. ダッシュボードで「Add New...」→「Project」を選択する
3. 「Import Git Repository」から、招待されたリポジトリ(customer-crm)を選んでインポートする
4. プロジェクトの「Storage」タブから「Connect Database」→「Neon」を選び、案内に従って接続する(あなた専用の新しいデータベースが自動的に作られます。元のシステムとはデータベースが完全に分かれるため、データが混ざることはありません)
5. プロジェクトの「Settings」→「Environment Variables」で、キーに `AUTH_SECRET` を追加する。値は以下の方法で自分で作成してください:
   - ブラウザで何らかのページを開いた状態でキーボードの F12(またはCmd+Option+I)を押し、「Console」タブを開く
   - `crypto.randomUUID()` と入力してEnterを押す
   - 表示された文字列をコピーして、`AUTH_SECRET` の値として貼り付ける
   - Production・Preview・Developmentの全てにチェックを入れて保存する
6. 「Deploy」を押してデプロイが完了するのを待つ
7. 完了したら表示されたURL(`◯◯◯.vercel.app`)にアクセスする。自動的にセットアップ画面に案内されるので、氏名・メールアドレス・パスワードを入力して自分の管理者アカウントを作成する
8. 作成が完了すると自動的にログインされ、そのまま顧客管理画面が使えます

## 今後について

- 開発者が機能を追加・修正した場合、あなたが何か操作をしなくても自動的にあなたの環境にも反映されます
- 保存されるデータは完全にあなた専用のもので、開発者や他の人と共有されることはありません
- 開発者もあなたのパスワードを知ることはできません

## トラブルシューティング(開発者向け)

- セットアップ画面(`/setup`)は最初の1回しか使えません。一度アカウントが作成された後は自動的に無効化されます
- 友人がパスワードを忘れた場合など、開発者側で手動でアカウントを作り直す必要が生じたときは、そのデプロイの環境変数(`DATABASE_URL`など)を `vercel env pull .env.local` で取得した上で、`npm run create-user -- "氏名" "メールアドレス" "パスワード"` を実行することで対応できます(この場合は開発者がパスワードを知ることになるため、あくまで最終手段です)
